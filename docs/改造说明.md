### 核心任务

#### 任务1：将项目中所有Sass/SCSS代码替换为纯CSS（适配移动端）
1. 替换规则：
   - Sass变量（$primary-color: #1677ff;）→ CSS自定义属性（--primary-color: #1677ff;），在根组件/App.vue中统一声明
   - Sass嵌套样式 → 解嵌套为标准CSS选择器（如 .header .title → .header-title）
   - Sass混合器（@mixin flex-center）→ 提取为CSS通用类（.flex-center { display: flex; align-items: center; justify-content: center; }）
   - Sass导入（@import './variables.scss'）→ 合并到对应CSS文件中
2. 移动端适配要求：
   - 移除Sass的响应式混合器后，用**CSS媒体查询**实现响应式（适配320px-768px移动端宽度）
   - 所有尺寸单位优先使用**rem/vw**（而非固定px），根字体大小设置为10vw（最大16px）
   - 简历组件的间距、字体大小、图片尺寸需随屏幕宽度自适应，避免移动端横向滚动
   - 保留原PC端样式效果，同时保证移动端视觉统一、操作便捷

#### 任务2：新增简历配置页面（Config.vue），全适配移动端
1. 页面功能：
   - 布局：
     ✅ PC端：左侧配置表单（占40%宽度），右侧实时预览简历（占60%宽度）
     ✅ 移动端（≤768px）：自动切换为上下布局（配置表单在上，预览区在下），适配竖屏展示
   - 可配置项：
     ✅ 基础信息：姓名、头像URL、求职意向、联系方式（电话/邮箱）
     ✅ 样式配置：主题色（对应CSS自定义属性--primary-color）、字体大小（大/中/小）、简历排版（紧凑/宽松）
     ✅ 内容开关：是否显示实习经历、项目经验、技能模块
   - 交互要求：
     ✅ 表单修改后实时更新预览（无需刷新），配置数据存入localStorage（刷新保留）
     ✅ 提供「重置默认配置」「保存配置」按钮，移动端按钮宽度100%，点击区域≥44px（符合移动端交互规范）
     ✅ 表单控件适配移动端：输入框高度≥40px，单选/复选框放大，避免误触
2. 技术要求：
   - 使用Vue3 Setup语法糖，ref/reactive管理状态，无需引入第三方UI库（保持轻量）
